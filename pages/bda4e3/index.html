流程控制

流程控制主要包括选择和循环。

选择就是if else 和switch。

选择

选择没什么好讲的，大致上就是if和switch，对于什么时候使用哪种结构？细讲的话，可以讲很多，不细讲也可以，因为代码写的 多了，就信手拈来。细讲，代码写的少，反而不理解。

if

情景一：在某种情况下执行

int a = 1;
if(a ==1 ){
	a++;
}
System.out.print(a);

# switch

关于switch就更简单了，确定情况是某几个值时，使用switch方便点，注意一点default和case break这些关键字就行了。

switch对于固定的值方便点，对于区间，也是可以的，比如分数（满分100）， 优秀及格什么的，确实值很多，但是把分数/10不就能把区间变成固定的值了吗。

int score = 59;
String lev；
switch (score/10){
	case 10:
		lev="满分";
		break;
	case 9:
		lev="优秀";
		break;
	...
	case 6:
		lev = "及格";
		break;
	default
		lev = "不及格"；
		break;
}

# 选择嵌套

选择嵌套其实还是熟练，自己领悟。

**if( xxx )**就是一个拦截条件，满足的在{}里，不满足的在{}外或者是else的{}；里，注意层级和嵌套关系，多练习即可。

# 循环

while、do while、for之间可以相互转换，有些时候转换会比较麻烦一点。

不知道要循环多少次时使用while和do while,知道固定次数使用for更简单点。

# while

适合不知道要循环多少次。

boolean 累不累 = true;
while( 累不累 ){
	睡一个小时;
	...
	// 这里累不累这个变量可能会发生改变
}

# do white

do while就比较特殊了，while在循环前会判断一次，如果这个时候判断条件的结果是false,那循环体是一次也不会执行的。

而do while则至少执行一次。

除此之外，遇到循环条件在循环前并不知晓，执行一次之后才知道循环条件，这时使用do while好点

do{
	考试
}while(考试不及格？)

# for

for循环适合固定循环次数的循环。

简单的例子，计算1+2+3...+99+100

使用数学公式的话

int result = (1+100)*100/2;
1

不优化，使用循环的话

int result = 0;
for(int i =1;i<=100;i++){
	result += i;
}

# break & continue

break和continue很简单， break是真的停止循环，continue是“继续”的意思。

就像在操场上跑步，break，是跑不动了，不跑了。

continue是跑不动了，回到起点，歇一会，继续跑。
