# 删除集合

删除集合 collection.drop(options)

删除集合及其所有索引和数据。为了删除系统集合，必须指定属性isSystem设置为true的options对象。

禁止将集合拖放到群集中，该群集是在其他集合中共享的原型。为了能够删除这样的集合，必须首先删除所有从属集合。

例子

arangosh> col = db.example; 
arangosh> col.drop（）; 
arangosh> col;
1

[ArangoCollection 76086, "example" (type document, status loaded)]
[ArangoCollection 76086, "example" (type document, status deleted)]

arangosh> col = db._example; 
arangosh> col.drop（{ isSystem：true }）; 
arangosh> col;
1
2
3
[ArangoCollection 76093, "_example" (type document, status loaded)]
[ArangoCollection 76093, "_example" (type document, status deleted)]
1

# 压实（压缩）

在v3.4.5中引入

压缩集合的数据 collection.compact()

压缩集合的数据以回收磁盘空间。对于MMFiles存储引擎，该操作将重置集合的最后压缩时间戳记，因此它将成为压缩的候选对象。对于RocksDB存储引擎，该操作将通过重写基础.sst文件并仅保留相关条目来压缩文档和索引数据。

在正常情况下，无需执行压缩操作，因为无论如何最终都会压缩集合数据。但是，在某些情况下，例如，在运行了大量更新/替换或删除操作之后，集合的磁盘数据可能包含许多过时的数据，必须为其回收空间。在这种情况下，可以使用压实操作。

# 特性

获取或设置集合的属性collection.properties()

返回包含所有集合属性的对象。

  • waitForSync：如果为*true，*则仅在将数据同步到磁盘后才创建文档。

  • keyOptions

    （可选），用于生成密钥的其他选项。这是一个包含以下属性的JSON数组（注意：某些属性是可选的）：

    • type：用于集合的密钥生成器的类型。
    • allowUserKeys：如果设置为true，则允许在文档的*_key属性中提供自己的键值。如果设置为 false，那么密钥生成器将仅负责生成密钥，并且在文档的_key*属性中提供自己的密钥值被认为是错误。
    • increment：自动增量密钥生成器的增量值。不适用于其他密钥生成器类型。
    • offset：自动增量密钥生成器的初始偏移值。不适用于其他密钥生成器类型。

  • schema（可选，默认为null）：指定文档的收集级别文档架构的对象。属性键rule，level并且message必须遵循文档架构验证 (opens new window)中记录 (opens new window)的规则

在群集设置中，结果还将包含以下属性：

  • numberOfShards：集合的分片数。
  • shardKeys：包含用于确定文档目标分片的文档属性的名称。
  • replicationFactor：确定每个分片在不同的DB服务器上保留多少个副本。必须在1-10的范围内，或者"satellite"是SatelliteCollection的字符串 （仅企业版）。 （仅集群）
  • writeConcern：确定在不同的DB服务器上同步每个分片需要多少个副本。如果集群中的副本数量少于这些副本，则分片将拒绝写入。但是，具有足够最新副本的分片写入将同时成功。writeConcern的值 不能大于ReplicationFactor。（仅集群）
  • shardingStrategy：为集合选择的分片策略。此属性将仅在群集模式下填充，而不会在单服务器模式下填充。（仅集群）
collection.properties(properties)
1

更改集合属性。属性必须是具有以下一个或多个属性的对象：

  • waitForSync：如果为*true，*则仅在将数据同步到磁盘后才创建文档。
  • plicationFactor：更改保留在不同数据库服务器上的分片副本的数量。有效值是1到10范围内的整数，或者"satellite"是SatelliteCollection的字符串（仅企业版）。 （仅集群）
  • writeConcern：更改在不同的DB服务器上同步每个分片需要多少个副本。如果集群中的副本数量少于这些副本，则分片将拒绝写入。但是，具有足够最新副本的分片写入将同时成功。writeConcern的值 不能大于ReplicationFactor。（仅集群）

注意：创建集合后，就不能再更改其他一些集合属性，例如type， keyOptions，numberOfShards或shardingStrategy。

# 数字

返回集合的数字 collection.figures(details)

返回一个对象，其中包含有关集合的统计信息。

设置details为，true将使扩展存储引擎特定的详细信息返回到图中（在v3.8.0中引入）。这些详细信息旨在调试ArangoDB本身，其格式可能会发生更改。默认情况下， details设置为false，因此不返回任何详细信息，并且其行为与ArangoDB的早期版本相同。

  • index.count：为集合定义的索引总数，包括预定义的索引（例如主索引）。
  • index.size：为索引分配的总内存（以字节为单位）。

  • cacheInUse
  • 缓存大小
  • cacheUsage

# GetResponsibleShard

返回给定文档的责任分片。 collection.getResponsibleShard(document)

返回带有负责的分片的ID的字符串。请注意，返回的分片ID是文档的分片键值的负责分片的ID，即使不存在此类文档，也将返回该分片ID。

该getResponsibleShard()方法只能在集群中的协调器上使用。

# 碎片

返回集合的可用分片。 collection.shards(details)

如果details未设置或将其设置为false，则返回一个数组，其中包含集合的可用分片的名称。

如果details设置为true，则返回一个以分片名称作为对象属性键的对象，将负责的服务器作为映射到每个分片属性键的数组返回。

领导者分片始终在负责任的服务器阵列中排在第一位。

该shards()方法只能在集群中的协调器上使用。

# 修订

返回集合的修订版ID collection.revision()

返回集合的修订版ID

修改文档数据时，可以通过在其中插入，删除，更新或替换文档来更新版本ID。

客户端可以使用集合的修订版本ID来检查集合中的数据是否已更改，或者自从先前获取修订版本ID以来是否仍未修改。

返回的修订版ID是一个字符串值。客户应将此值视为不透明字符串，并且只能将其用于相等/不相等比较。

# 路径

返回集合的物理路径 collection.path()

该路径操作返回与用于收集数据的物理存储路径的字符串。

该path()方法将在群集中不返回任何有意义的东西。在单服务器ArangoDB中，此方法将仅返回MMFiles存储引擎的有意义的数据。

# 校验和

计算集合中数据的校验和 collection.checksum(withRevisions, withData)

该校验和运算，计算对包含集合中的所有文档键的总哈希值集合。

如果withRevisions的可选参数设置为true，则文档的修订版ID也将包含在哈希计算中。

如果可选参数withData设置为true，那么还将对所有用户定义的文档属性进行校验和。将文件数据包括在校验和中将使计算速度变慢，但更准确。

校验和计算算法在ArangoDB 3.0中发生了变化，因此3.0和更早版本对同一数据的校验和将有所不同。

# 卸载集合

卸载集合 collection.unload()

开始从内存中卸载集合。请注意，卸载将推迟到所有查询完成后再进行。

群集集合无法卸载。

例子

arangosh> col = db.example; 
arangosh> col.unload（）; 
arangosh> col;
1
2
3
[ArangoCollection 65552, "example" (type document, status loaded)]
[ArangoCollection 65552, "example" (type document, status unloaded)]
1
2

# 重命名

重命名集合 collection.rename(new-name)

使用new-name重命名集合。该新名称不能已被用于不同的集合。new-name也必须是有效的集合名称。有关有效集合名称的更多信息，请参考命名约定 (opens new window)。

如果重命名由于任何原因而失败，则会引发错误。如果重命名该集合成功，则该集合还将_graphs在当前数据库中该集合内的所有图形定义中重命名。

该rename()方法不能在群集中使用。

例子

arangosh> c = db.example;
arangosh> c.rename("better-example");
arangosh> c;
1
2
3
[ArangoCollection 76238, "example" (type document, status loaded)]
[ArangoCollection 76238, "better-example" (type document, status loaded)]
1
2
