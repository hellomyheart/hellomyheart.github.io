java核心类

把面向对象学完，Java的语法基础就已经学完了。

后面的东西大多数都是别人写好的一些类。

# String

比如String，String的底层是Char数组(新版本的jdk变成了byte数组)，我们自己也可以写一个类似String功能的类。

但是这样，每个人写的String肯定有不同的地方。Java核心类就是Java官方写的一些“通用工具”类，方便我们进行Java开发。

换句话说，Java官方把开发过程中常用到的一些东西，写成类，放到jdk中，方便开发人员使用。

String是开发过程中经常使用的数据类型。

在设计String这个类的时候，基于安全性的考虑，将String设置成了不可变的。String一旦实例化，这个对象就是不可变的，如果修改这个对象，其实会创建一个新的对象。

这样安全了，但是对String的修改操作会降低Java程序的性能。

String s = "";
for (int i = 0; i < 1000; i++) {
    s = s + "," + i;
}

这时会有1000个String对象，但是我们需要的只是最后一个对象，中间对象的创建，会消耗大量内存，并且之后又需要GC回收。

String对象从实例化开始，直到String对象被GC回收。都是保持不变的。

假如需要对String做大量修改操作，每一次修改操作都会创建一个String实例，消耗内存，影响GC效率。

为此JDK设计了StringBuffer和StringBuilder

# StringBuffer&StringBuilder

String适合保存（存储）字符串，这种字符串应该是不经常变动的 最终值。

而StringBuffer&StringBulider则适合做字符串的修改操作。

对StringBuffer对象的修改操作，都是对该对象进行的修改 （底层是数组），不会创建新的对象。

append方法是追加，在字符串后面加入 新的内容，相当于 Sting 的 +连接。

StringBuffer还支持链式调用。

StringBuffer还支持insert（插入）方法，delete(删除)方法等。

StringBuffer在设计时考虑到了线程安全，但是这样性能很低，并且很多开发场景是不需要考虑StringBuffer的线程安全的。

于是有了StringBuilder,将线程安全去掉，提高了运行速度。

如果你的String变量需要在多个线程间调用、修改,那么使用StringBuffer好点，如果不是的话，使用StringBuilder更好。

如果是格式化的拼接字符串，比如 123,456,789,012,156,325，元素与元素直接使用，分割，我们使用StringBuilder提高了Java性能,但是我们要做很多处理。

public class Main {
    public static void main(String[] args) {
        String[] names = {"Bob", "Alice", "Grace"};
        var sb = new StringBuilder();
        sb.append("Hello ");
        for (String name : names) {
            sb.append(name).append(", ");
        }
        // 注意去掉最后的", ":
        sb.delete(sb.length() - 2, sb.length());
        sb.append("!");
        System.out.println(sb.toString());
    }
}

# StringJoiner

public class Main {
    public static void main(String[] args) {
        String[] names = {"Bob", "Alice", "Grace"};
        var sj = new StringJoiner(", ");
        for (String name : names) {
            sj.add(name);
        }
        System.out.println(sj.toString());
    }
}

StringJoiner底层是StringBuilder，封装了一下，方便我们进行格式化拼接字符串。

同时String也有一个String.join()静态方法，方便拼接后生成一个String字符串，（没有提供开头和结尾设置）。

一切都是为了更好的开发。

至于后面的包装类，最早是为了后面集合类型、泛型这些不能使用基本数据类型的一种方案。

包装类出现后，集合里也可以放入基本类型对应的包装类型元素，使用泛型的地方，也可以使用包装类，将泛型引用到基本类型（的包装类）和引用类型。

当然，为了方便开发，基本类型和对应包装类之间的转换做了优化，自动装箱（基本类型 -> 包装类型）、自动拆箱（包装类型 -> 基本类型）可以无感操作。

当然，包装类是类，就可以写一些帮助方法，方便操作。比如常见的类型转换等等。

# JavaBean

JavaBean就是属性私有，暴露getter和setter，这里最好使用ide一键生成getter和setter方法，（或者使用和Lombok类似的注解）

后面很多第三方帮助类，对JavaBean进行复制、设置时，都会使用反射来调用getter、setter方法，生成getter和setter方法更不容易出错。

# 枚举

枚举是对常量类的优化，同时支持编译检查。枚举具有特殊性，也具有普通类的特性。

# 枚举特殊性

枚举类中定义的枚举值是单例模式（全局唯一），这个对象（枚举值）的类型就是这个枚举类。

enum Weekday {
    SUN, MON, TUE, WED, THU, FRI, SAT;
}
1
2
3

SUN是一个枚举对象，他的类型是Weekday。

因为枚举对象是全局唯一，所以可以使用 == 进行比较。

枚举不可以使用 new 实例化,这些都是特殊性。

# 枚举普通性

枚举是类，也可以定义变量，虽然枚举不可以使用new,但可以写构造函数，在类内部定义枚举时调用。

枚举是类，所以也是继承自Object,肯定也有toString方法，也可以重写。

enum Weekday {
    MON(1, "星期一"), TUE(2, "星期二"), WED(3, "星期三"), THU(4, "星期四"), FRI(5, "星期五"), SAT(6, "星期六"), SUN(0, "星期日");

    public final int dayValue;
    private final String chinese;

    private Weekday(int dayValue, String chinese) {
        this.dayValue = dayValue;
        this.chinese = chinese;
    }

    @Override
    public String toString() {
        return this.chinese;
    }
}

# 记录类

记录类是jdk14支持的特性。

记录类就是为了方便写不变类。改了编译器，可以使用记录类的语法写不变类。

# BigInteger&BigDecimal

这两个类是为了解决精度问题。整数有最大限制，浮点数精度不是非常高。

# BigInteger

BigInteger中的计算都是通过方法来实现的，使用BigInteger对象进行计算可以进行大数的计算，保证精确度，返回值是一个新的BigInteger对象。

BigInteger也可以转换成Long(不太大的时候)

如果太大，就只能toString了。

# BigDecimal

和BigInteger类型，底层是BigInteger和一个小数点位数。

BigDecimal注意一下舍入模式、compareTo()和equals()的不同之处就可以了。

# Math

Math里面封装了常用的数学函数和随机数。

Math.random()会随机生成一个[0,1)区间的小数。把这个小数理解成百分比的话，可以生成任意区间的数字了。

// 区间在[MIN, MAX)的随机数
public class Main {
    public static void main(String[] args) {
        double x = Math.random(); // x的范围是[0,1)
        double min = 10;
        double max = 50;
        double y = x * (max - min) + min; // y的范围是[10,50)
        long n = (long) y; // n的范围是[10,50)的整数
        System.out.println(y);
        System.out.println(n);
    }
}
