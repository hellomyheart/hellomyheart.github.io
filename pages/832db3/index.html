    数组排序

    对数组进行排序是程序中非常基本的需求。常用的排序算法有冒泡排序、插入排序和快速排序等。

    对于小白，如果以下内容不理解可以不必深究。学会循环就好了，多练练简单的循环就好了，时间久了，这些你会明白的，不要心急。

    我们来看一下如何使用冒泡排序算法对一个整型数组从小到大进行排序：

    import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        int[] ns = { 28, 12, 89, 73, 65, 18, 96, 50, 8, 36 };
        // 排序前:
        System.out.println(Arrays.toString(ns));
        for (int i = 0; i < ns.length - 1; i++) {
            for (int j = 0; j < ns.length - i - 1; j++) {
                if (ns[j] > ns[j+1]) {
                    // 交换ns[j]和ns[j+1]:
                    int tmp = ns[j];
                    ns[j] = ns[j+1];
                    ns[j+1] = tmp;
                }
            }
        }
        // 排序后:
        System.out.println(Arrays.toString(ns));
    }
}

    冒泡排序的特点是，每一轮循环后，最大的一个数被交换到末尾，因此，下一轮循环就可以“刨除”最后的数，每一轮循环都比上一轮循环的结束位置靠前一位。

    冒泡排序，小数靠前，大数沉底，两两相邻比较。

    很朴素的思想，最简单的排序是两两比较。即使有多个数，也让他们两两相互比较，这样的话，排序思路很简单，可能效率不高，但是简单。容易执行（在代码里面就是容易写代码）

    第一遍冒泡排序，大的往后沉底，最后一个是最大的，不需要再排序了。

    第二遍冒泡排序，可以少比较一个，同时，那个也是最大的数（下标也最大）。

    第三遍冒泡排序，可以少比较两个。

    所以最外层循环是控制遍数,一遍确定一个最大的数。所以length-1遍就能确定length-1个数，剩下的那个就是最小 的数，下标为1.

    同时每一遍的比较次数与遍数有关。（本质是比较次数与少比较个数有关，少比较个数与遍数有关），即正常比较次数（length-1）减去少比较次数（与遍数相等）i,变成了length-1-i.

    结合for循环特性，就可以写冒泡循环了（而不是简单的背冒泡循环）

    另外，注意到交换两个变量的值必须借助一个临时变量。像这么写是错误的：

    这样可以不需要临时变量

    实际上，Java的标准库已经内置了排序功能，我们只需要调用JDK提供的Arrays.sort()就可以排序：

    import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        int[] ns = { 28, 12, 89, 73, 65, 18, 96, 50, 8, 36 };
        Arrays.sort(ns);
        System.out.println(Arrays.toString(ns));
    }
}

    # 小结

    常用的排序算法有冒泡排序、插入排序和快速排序等；

    冒泡排序使用两层for循环实现排序；

    交换两个变量的值需要借助一个临时变量。

    可以直接使用Java标准库提供的Arrays.sort()进行排序；

    对数组排序会直接修改数组本身。
